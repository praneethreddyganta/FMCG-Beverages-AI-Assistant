Solution Architecture

Natural Language to SQL

System prompts inject the SQLite schema description into Gemini to translate natural queries into valid SELECT queries dynamically.

SQLite Execution Engine

Express server runs the validated query, compiles a spreadsheet grid, and auto-calculates horizontal bar chart values.

Dynamic Orchestration Flow

1. User Question in UI
2. LLM SQLite Query Translation
3. Database Execution & Data Fetch
4. LLM Narrative Summary Formulation
5. Tabular & Visual Chart Displays

Simulated FMCG Data Sourcing

  • Scale & granularity: Models 15 products across 5 categories, 24 stores, 4 regions, and 16 weeks of weekly sales and inventory history (7,680 records).
  • Coherent Stock Logic: Models structural inventory dependencies: closing_stock = opening + received - sold.
Promotion Multipliers

Stockout Modeling

Forces delivery delay flags (units received = 0) to verify stockout reasoning and inventory warning outputs.

Risk Control & Future Roadmap

Current System Failure Points
  • Breaks on vague requests without numeric parameters.
  • File-based SQLite locks under concurrent write queries.
Version 2.0 Enhancements
  • Semantic Cache: Map similar prompts to SQL to reduce LLM costs.

  • Multi-Agent Verification: Add verification loop for SQL aggregation safety.
