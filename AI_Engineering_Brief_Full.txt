AI Engineering
Assessment Brief
Consumer Goods (FMCG)  ·  Beverages Category  ·  Data & AI Team
Context
You are part of the Data & AI team at a Consumer Goods (FMCG) organisation, working on the Beverages category. Business users across functions frequently request:

Promotional performance summaries
Inventory movement insights
Regional sales comparisons
Product-level campaign impact

Currently, these requests are handled through manual dashboarding and ad-hoc analysis by data teams — leading to delayed decision-making, repetitive analysis, and heavy dependency on analysts.

Leadership wants to explore whether an AI-powered assistant can help business users obtain such insights conversationally — without needing to involve the data team for every request.

What We Provide

Brief

Everything else
Environment setup, data sourcing, tooling, and infrastructure are entirely your responsibility.

You are expected to source or generate your own dataset, set up your own environment, choose your own tools, and build your solution end-to-end. This is intentional — how you handle the open-ended setup is part of the evaluation.

Suggested Dataset Structure
To help you get started, below is a guide to the kinds of data your AI assistant should be able to reason over. You do not need to use this exact structure — treat it as a reference for what realistic FMCG data looks like. All data can be synthetically generated.

1.  Sales & Promotions
This is the core dataset. It captures weekly sales performance alongside any active promotional activity. Most of the analytical questions the assistant should handle will be rooted here.

Field
Type
Notes
week_start_date
Date
ISO format. Weekly granularity, e.g. 2024-01-01, 2024-01-08.
product_id
String (FK)
Foreign key linking to Product Master.
store_id
String (FK)
Foreign key linking to Store Master.
region
String
e.g. North, South, East, West. Can be denormalised here for convenience.
units_sold
Integer
Total units sold in the week. Should vary meaningfully with promotion flag.
revenue
Decimal
Weekly revenue in local currency. Derived from units × unit price × (1 − discount).
promotion_flag
Boolean
True if any promotion was active for this product/store in this week.
promotion_type
String
e.g. Price Cut, BOGO, Display Feature, Bundle. Null when no promotion.
discount_pct
Decimal (0–1)
Proportion of price discounted. Use 0 when no promotion is active.

2.  Inventory
Tracks stock movement at a weekly, store-product level. Enables questions about stockouts, over-stocking, and whether promotions led to inventory drawdown.

Field
Type
Notes
week_start_date
Date
Must align to the same weekly grain as Sales & Promotions.
product_id
String (FK)
Foreign key linking to Product Master.
store_id
String (FK)
Foreign key linking to Store Master.
opening_stock
Integer
Units on hand at the start of the week.
units_received
Integer
Units restocked or delivered during the week.
units_sold
Integer
Should match units_sold in the Sales table for the same key.
closing_stock
Integer
Calculated: opening_stock + units_received − units_sold.
stockout_flag
Boolean
True if closing_stock reached zero at any point during the week.

3.  Product Master
A reference table for all beverages in the dataset. Allows the assistant to resolve product names, filter by category, and apply correct pricing.

Notes

Parent brand name.
category

e.g. Carbonated, Juice, Water, Energy, Dairy.
sub_category
String
e.g. Sparkling Water, Fruit Juice, Sports Drink.
pack_size_ml
Integer
Volume in millilitres, e.g. 250, 500, 1000.
unit_price
Decimal
Standard retail price before any discount.

4.  Store Master
A reference table for retail locations. Enables regional filtering, store-type breakdowns, and geographic comparisons.

Field
Type
Notes
store_id
String (PK)
Unique identifier, e.g. STR-042.
store_name
String
e.g. FreshMart Central, QuickStop Southside.
region
String
High-level grouping: North, South, East, West.
city
String
City or town name.
store_format
String
e.g. Supermarket, Hypermarket, Convenience, Wholesale.

Tip for simulation: Aim for at least 4 regions, 10–20 products across 3–5 categories, 20–50 stores, and 12–24 weeks of history. Make promotions visible in the data — units sold should be noticeably higher in promotion weeks to give the AI something meaningful to detect
